Introducción

Todo proyecto nace de una necesidad real. BloomStock nació del caos de hojas de cálculo y la certeza de que existía una solución más elegante.

La idea de construir un Sistema de Punto de venta y de Inventario surge de una necesidad concreta: muchas pequeñas empresas y emprendedores gestionan su stock con hojas de cálculo desactualizadas, procesos manuales y errores costosos. Observar esa fricción cotidiana fue la chispa que encendió este proyecto.

Más allá de resolver un problema real, este proyecto es una oportunidad para poner en práctica mis habilidades como desarrollador de software, explorar arquitecturas modernas y documentar todo el proceso de construcción desde cero hasta un producto funcional.

El objetivo general de BloomStock es ofrecer una plataforma web intuitiva que permita a cualquier negocio —sin importar su tamaño— registrar, consultar y gestionar su inventario de manera ágil, con reportes claros y un acceso seguro multiusuario.

✅

Funcionalidades Esperadas y Alcance

La primera versión del sistema —el Producto Mínimo Viable— estará enfocada en las funcionalidades esenciales que aporten valor inmediato al usuario:

  • 📦 Gestión de Productos (CRUD completo) Crear, consultar, editar y eliminar productos con nombre, descripción, categoría, precio y unidad de medida.
  • 📊 Control de Stock (Entradas y Salidas) Registro de movimientos de inventario con fecha, cantidad, tipo de operación y usuario responsable.
  • 🔔 Alertas de Stock Mínimo Notificaciones visuales cuando un producto cae por debajo del umbral configurado, evitando quiebres de stock.
  • 📈 Generación de Reportes Básicos Resumen del estado actual del inventario, histórico de movimientos y productos con bajo stock, exportables a PDF.
  • 🎨 Interfaz de Usuario Intuitiva Diseño limpio, responsivo y accesible que funcione correctamente en dispositivos móviles y escritorio.
  • 🔐 Autenticación y Login Seguro Sistema de registro e inicio de sesión con tokens JWT, roles de usuario (administrador / operario) y cierre de sesión seguro.
  • 🗂️ Gestión de Categorías Organización de productos por categorías personalizables para facilitar la búsqueda y el filtrado.

Objetivos de Aprendizaje Personal

Mi objetivo es poder aprender de todo para tener mas conocimientos, tener crecimiento personal, desarrollar un proyecto y que de sus frutos,pero mas que nada desarollar mis habilidades de trabajo en equipo y individual al igual que las softskills para tener mas oportunidades en un futuro ydesarrollar mas proyectos en un futuro.

  1. 🗄️ Diseño de Bases de Datos Relacionales (SQL) Modelado entidad-relación, normalización, índices, relaciones y buenas prácticas con MySQL / PostgreSQL.
  2. 🏗️ Arquitectura de Software (Patrón MVC) Separación clara de responsabilidades entre Modelo, Vista y Controlador para construir código escalable y mantenible.
  3. ⚛️ Desarrollo Frontend con React Componentes reutilizables, hooks, manejo de estado global con Context API o Redux, y consumo de APIs REST.
  4. ☕ Desarrollo Backend con Java / Spring Boot Creación de APIs RESTful robustas, manejo de dependencias con Maven, seguridad con Spring Security y JWT.
  5. 🐙 Uso Avanzado de Git y GitHub Estrategias de branching (Git Flow), pull requests, resolución de conflictos, y documentación del repositorio.
  6. ☁️ Implementación en la Nube (AWS / Railway) Despliegue del backend en un servidor en la nube, configuración de variables de entorno y dominio personalizado.
  7. ⏱️ Gestión del Tiempo y Metodología Personal Uso de Kanban personal (GitHub Projects / Notion) para planificar sprints, estimar tareas y mantener el ritmo.
  8. 📝 Documentación Técnica Profesional Redacción de READMEs, docstrings, comentarios de código y documentación de API con Swagger/OpenAPI.
  9. 🧩 Pensamiento Lógico y Resolución de Problemas Enfrentar bugs reales, decisiones de diseño con trade-offs y aprender a buscar soluciones de forma metódica.
